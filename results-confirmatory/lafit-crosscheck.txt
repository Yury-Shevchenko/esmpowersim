PowerAnalysisIL cross-check (validation check i, prereg §7 / manuscript §3.5, §4.1)
====================================================================================

Purpose
  Confirm that at the balanced reduction (CV -> 0, no cap, full compliance,
  exogenous triggering, fixed occasion count, no autoregression) the triggered
  engine reproduces the field-standard fixed-count power tool, PowerAnalysisIL
  (Lafit, Adolf, Dejonckheere, Myin-Germeys, Viechtbauer & Ceulemans, 2021),
  to within Monte Carlo error. This is what entitles the paper to describe the
  triggered framework as a strict superset rather than a re-implementation.

Provenance
  Date run          2026-07-20
  Our engine        esmpowersim R/ (seeds, models, dgm, fit, performance),
                    Model 3 (within-person random slope)
  Reference         PowerAnalysisIL, github.com/ginettelafit/PowerAnalysisIL
                    commit 11ec9e12c1ef8f38f928fc9c1dc78cf4f18966a9
                    (R/Sim.Data.IL.R + R/Fit.model.IL.R, Model == 3),
                    run VERBATIM (nlme::lme(Y ~ X, random = ~1 + X | subjno,
                    correlation = NULL, method = "REML"))
  Reproduce         Rscript tools/lafit-crosscheck.R   (needs network, nlme,
                    lme4, MASS; fetches the two Lafit files at run time)
  Replications      R = 800   seed = 20260709

Matched design
  Model 3; fixed occasion count T = 42; no AR (phi = rho = 0); exogenous
  X ~ N(0,1); sd_b0 = 1, sd_b1 = 0.3, rho_b = 0, sigma_e = 1. Power = share of
  two-sided Wald/t tests of the within-person effect significant at alpha = .05.

Result  (power; MCSE ~ .016-.018; 100% convergence on both sides)

  cell                    PowerAnalysisIL   ours (engine)   ours (like-for-like)
  N=60  T=42  beta1=0.08       0.431           0.444              0.490
  N=60  T=42  beta1=0.11       0.682           0.696              0.704
  N=30  T=42  beta1=0.11       0.463           0.449              0.417

  ours (engine)        = esmpowersim as shipped: lme4 y ~ x + y_lag + (1 + x | id)
  ours (like-for-like) = our DGM + a PowerAnalysisIL-equivalent fit with no y_lag
                         (lme4 y ~ x + (1 + x | id)), isolating the effect of our
                         lagged-outcome control

Verdict: PASS.
  Engine vs PowerAnalysisIL: deviations +0.013, +0.014, -0.014 — all below one
  MCSE. The like-for-like variant agrees within ~1-2 MCSE (wider spread is
  independent-stream Monte Carlo noise, not a systematic gap). The engine
  reproduces PowerAnalysisIL power to within Monte Carlo error, meeting the
  pre-specified criterion in §3.5.

Note on the model correspondence
  PowerAnalysisIL's Model 3 carries temporal dependence as AR(1) ERRORS (nlme
  corAR1); this engine carries it as a lagged OUTCOME (lme4 + y_lag), matching
  Lafit's own Models 9-11. The two coincide exactly only with no autoregression,
  which is the configuration cross-checked here. See R/models.R.
